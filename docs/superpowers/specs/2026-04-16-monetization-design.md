# Dobby MVP 商業化策略設計

**日期：** 2026-04-16  
**狀態：** 已核准  

---

## 背景

Dobby 是一個 Windows 桌面應用（Wails + Go + React），讓個人用戶設定規則來自動監視資料夾、重新命名並移動檔案。MVP 商業化目標是**盡快產生第一筆收入**，驗證有人願意為這個問題付費。

---

## 目標用戶

個人用戶（Windows），需要自動整理下載資料夾、發票、合約、照片等檔案。

---

## 商業模式：買斷制 + 免費試用

### 定價

- **$9.99 USD**（約 NT$320）
- 一次付費，永久使用
- 低於競品 Hazel（Mac，$42），對 Windows 用戶有吸引力

### 試用機制

- 首次啟動提供 **14 天全功能免費試用**，無需帳號
- App 首次啟動時記錄日期於本地（混淆儲存，防止簡單刪檔重試）
- 試用到期後：規則設定保留，但背景處理器停止運作，UI 顯示購買提示
- 用戶輸入有效 License key 後恢復全功能

---

## 授權與付款

### 販售平台：Gumroad

- 個人開發者最快上線的平台（15 分鐘內可開賣）
- 自動處理信用卡、PayPal、Apple Pay
- 自動產生 License key 並寄送給買家
- 抽成約 10%

### License Key 機制

- 格式：`DOBBY-XXXX-XXXX-XXXX`
- 用戶在 Settings 頁面輸入 key 啟動
- MVP 階段：本地 hash 驗證（不需後端 server）
- 一個 key 綁定一台機器（machine ID）

### 技術範疇（MVP）

- 不做複雜防破解，只確保正常付費流程順暢
- 驗證邏輯：`hash(licenseKey + machineID)` 對比內建校驗碼

## 上市策略

### 發布順序

1. **Beta 測試**：邀請 5–10 位真實用戶（朋友、Discord），收集回饋並修 bug
2. **正式發布**：上 Product Hunt 衝榜（選週二或週三發布效果最佳）
3. **持續推廣**：在 Reddit、PTT、Dcard 分享

### 推廣渠道

| 渠道 | 目標 |
|------|------|
| Product Hunt | 英語市場首波曝光 |
| Reddit (`r/windows`, `r/productivity`, `r/datacurator`) | 精準受眾，軟體討論社群 |
| PTT 軟體版 / Dcard | 台灣市場，個人開發者貼文友善 |

## MVP 成功指標

| 指標 | 目標 |
|------|------|
| 試用下載人數 | 100 人 |
| 付費轉換率 | 5–10% |
| 付費人數 | 5–10 人 |
| 第一筆收入 | $50–$100 USD |

達到以上指標即驗證產品有市場，可進一步考慮調整定價或擴充功能。

## 待實作功能清單

1. **試用期計時機制** — 本地記錄首次啟動日期（混淆儲存）
2. **試用到期攔截** — 背景處理器啟動前檢查授權狀態
3. **License key 驗證邏輯** — hash 驗證 + machine ID 綁定
4. **Settings 頁面 License 輸入 UI** — 輸入框、驗證回饋、啟動成功狀態
5. **Gumroad 商品頁設定** — 上架、定價、License key 自動寄送
