# 檔案管家（File Manager）需求規格文件

**日期：** 2026-04-16
**版本：** v1.0

---

## 1. 問題背景

工作者在日常作業中會累積大量工作文件（報告、截圖、設計匯出檔、合約等），這些檔案常面臨兩個核心問題：

1. **命名無意義**：工具匯出的檔案預設命名（如 `2026-04-16-001.png`）缺乏語意，事後難以辨識內容。
2. **歸檔耗時**：即使有既有的分類系統，手動執行歸檔費時費力，導致檔案長期堆積。

這個問題不限於特定工具（Figma、截圖工具、瀏覽器下載等皆有此困擾）。

---

## 2. 目標

### 主要目標
- 根據使用者定義的規則，**自動命名**新進檔案
- 根據使用者定義的規則，**自動歸類**（移動至正確資料夾）新進檔案
- 提供直覺的 Web 介面，讓使用者管理規則與查看操作紀錄

### 次要目標
- 可作為個人效率工具，未來具備商業化潛力
- 系統架構清楚，方便日後加入 AI 輔助命名功能

### 不在範圍內（v1）
- AI 自動命名（不依賴外部 API）
- 雲端同步或跨裝置功能
- 行動裝置支援
- 多使用者 / 帳號系統

## 3. 使用者故事

| # | 身份 | 需求 | 目的 |
|---|------|------|------|
| 1 | 使用者 | 我想設定一個規則：「監控 `~/Downloads`，所有 `.png` 檔命名為 `{project}-{type}-{YYYY-MM-DD}-{seq}` 並移至 `~/Projects/design/`」 | 不需手動整理設計匯出圖 |

## 4. 核心功能

### 4.1 資料夾監控
- 使用者可指定一個或多個「監控資料夾」
- 系統常駐於背景，即時偵測新增或修改的檔案
- 每個監控資料夾可設定是否遞迴監控子資料夾

### 4.2 命名規則
使用者可定義命名樣版，支援以下變數：

| 變數 | 說明 | 範例 |
|------|------|------|
| `{project}` | 使用者定義的專案名稱 | `my-app` |
| `{type}` | 使用者定義的檔案類型標籤 | `screenshot`, `report` |
| `{YYYY}` | 西元年 | `2026` |
| `{MM}` | 月份（兩位數） | `04` |
| `{DD}` | 日期（兩位數） | `16` |
| `{seq}` | 當日序號（自動遞增） | `001` |
| `{original}` | 原始檔名（不含副檔名） | `Untitled` |
| `{ext}` | 副檔名 | `png` |

命名樣版範例：`{project}-{type}-{YYYY}{MM}{DD}-{seq}.{ext}`
產出結果範例：`my-app-screenshot-20260416-001.png`

### 4.3 分類規則（歸檔）
- 每條規則可指定一個目標資料夾
- 支援動態路徑（如 `~/Projects/{project}/assets/`）
- 若目標資料夾不存在，系統自動建立

### 4.4 規則篩選條件
每條規則可設定觸發條件（AND 邏輯）：
- 副檔名過濾（如只處理 `.png`, `.jpg`, `.pdf`）
- 來源資料夾（只對特定監控資料夾的檔案生效）
- 檔名包含關鍵字（選填）

### 4.5 操作紀錄
- 記錄每次自動操作：原始路徑、新路徑、套用規則、時間
- 支援查詢與篩選
- 保留最近 1000 筆紀錄（v1）

### 4.6 桌面通知
- 檔案被成功處理時，發送系統桌面通知
- 通知內容：原始檔名 → 新檔名

## 6. 資料模型（概念）

### Rule（規則）
```
id            唯一識別碼
name          規則名稱（人類可讀）
enabled       是否啟用
watch_folder  監控資料夾路徑
recursive     是否遞迴監控
filters       篩選條件（副檔名、關鍵字等）
name_template 命名樣版字串
target_folder 目標資料夾路徑
project       專案名稱（用於 {project} 變數）
type_label    類型標籤（用於 {type} 變數）
created_at    建立時間
updated_at    更新時間
```

### Log（操作紀錄）
```
id            唯一識別碼
rule_id       套用的規則 ID
original_path 原始檔案完整路徑
new_path      處理後檔案完整路徑
status        success / error
error_message 錯誤訊息（若失敗）
processed_at  處理時間
```

## 7. Web UI 頁面規劃

| 頁面 | 功能 |
|------|------|
| 首頁 / 儀表板 | 系統狀態、今日處理數量、最近操作紀錄 |
| 規則管理 | 新增、編輯、刪除、啟用/停用規則；顯示規則列表 |
| 操作紀錄 | 顯示歷史處理紀錄，支援依規則或時間篩選 |
| 設定 | 啟動/停止背景服務、通知設定 |

---

## 8. 建議技術棧

| 層 | 技術 |
|----|------|
| 桌面框架 | Wails v2（Go 後端 + 原生 WebView）|
| 檔案監控 | Go + `fsnotify` |
| 資料庫 | SQLite（via `modernc.org/sqlite`，純 Go，跨平台無需 CGO）|
| Web UI | React + Vite |
| 桌面通知 | Wails 內建通知 API |
| 部署 | 單一原生桌面執行檔，支援 Windows / macOS / Linux |

**跨平台 WebView 對應：**
- Windows：WebView2（Chromium-based）
- macOS：WKWebView（Safari-based）
- Linux：WebKitGTK

---

## 9. 成功指標

- 使用者能在 5 分鐘內完成第一條規則的設定
- 新進檔案在 3 秒內被自動處理
- 規則正確率達 100%（規則設定明確的前提下）
- 使用者不再需要手動命名或移動受監控資料夾內的檔案
